项目名称：基于SDN的工业控制网络智能流量监测与蜜罐协同防护系统

一、项目概述
本项目设计并实现一个面向工业控制场景的轻量级安全隔离系统，采用SDN三层架构：
- 应用层（安全服务层）：运行AI异常检测模型与Web交互界面，负责流量分析、决策与人机协同；
- 控制层（SDN控制器）：实现拓扑管理、流状态监控与流表策略下发；
- 数据层（网络设备层）：由支持OpenFlow的软件交换机组成，执行数据转发与流量重定向。

系统核心功能：对Modbus/TCP流量进行实时异常识别，并将可疑流量动态导向蜜罐服务，实现“检测-响应”闭环。项目不依赖真实工业设备，全程在Mininet仿真环境中实现，突出SDN可编程性与高级网络编程技术的综合应用。

二、关键问题回答
2.2 如何实现这个题目？（HOW）

（1）拓扑设计与软硬件支持
• 拓扑结构（Mininet仿真）：
    Web UI / AI服务（应用层） 
           ↓ REST/Event
    Ryu SDN控制器（控制层）
           ↓ OpenFlow 1.3
    +-----------------------------+
    |  Software Switch (OVS)      |
    |  +--------+   +----------+ |
    |  | PLC    |   | Honeypot | | ← 10.0.0.10:502（真实服务）与10.0.0.20:502（蜜罐）
    |  +--------+   +----------+ |
    |        ↑           ↑        |
    |  +----------+ +----------+ |
    |  | Client   | | Attacker | | ← 正常客户端与攻击者主机
    |  +----------+ +----------+ |
    +-----------------------------+

• 软硬件方案：
  - 仿真平台：Mininet（创建拓扑） + Open vSwitch（数据平面）；
  - 控制器：Ryu Framework（Python，支持REST API与事件驱动）；
  - 应用层：Flask（Web UI + AI服务API封装）+ Vue；
  - 蜜罐：Python + pymodbus（极简Modbus服务，记录攻击行为）；
  - 特点：零硬件依赖、全开源、部署便捷，适合课程项目开发与演示。

（2）流量生成策略
• 流量类型：
  - 正常流量：周期性Modbus读写请求（e.g., 每5秒读寄存器40001–40010），包率 < 10 pkt/s；
  - 异常流量：
    a) 扫描类：遍历功能码（0x01–0xFF）或寄存器地址；
    b) 洪泛类：高频写请求（>100 pkt/s）；
    c) 篡改类：向关键寄存器（如40001）写入非法值。
• 生成方式：
  - 正常流量：pymodbus.client.sync.ModbusTcpClient + 定时脚本；
  - 异常流量：scapy构造Modbus TCP包（ModbusADURequest + ModbusPDU子类）；
• 特征工程（供AI模型使用）：
  - 流级特征（避免包级高开销）：包率（pkt_rate）、字节率（byte_rate）、功能码熵（func_code_entropy）、目标寄存器地址标准差（reg_addr_std）。

• 数据采集技术：
  - 流量特征：控制器定期拉取OFPMP_FLOW_STATS，计算速率特征；
  - AI决策日志：Web UI调用/detect API时记录输入特征与输出标签；
  - 流表变更：Ryu监听OFPFlowStatsReply，存档流表变更历史；
  - 蜜罐交互：pymodbus服务日志记录攻击者IP、时间、操作详情。
• 可视化（加分项）：Grafana仪表盘实时展示拓扑状态、流量速率曲线、AI异常评分曲线、蜜罐告警列表。

三、项目特色与教学价值
• 端到端闭环：完整实现“流量生成→特征提取→AI检测→SDN响应→蜜罐捕获”全链路；
• 技术聚焦：避开复杂模型训练，突出SDN控制逻辑与高级编程（REST/异步/事件驱动）；
• 可演示性强：Wireshark对比（正常流 vs 重定向流）、Web UI实时告警、蜜罐交互日志；
• 工程思维：模块解耦（AI服务独立API）、可观测性设计（指标采集）、MVP路径清晰。
本项目不仅是SDN与高级网络编程知识的综合应用，更是对“智能网络”前沿趋势的一次高质量教学实践。
